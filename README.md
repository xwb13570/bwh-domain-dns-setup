## 搬瓦工域名解析是什么，为什么必须做这一步

**域名解析**，说白了就是告诉全球的 DNS 系统：这个域名，对应的是哪台服务器的 IP 地址。没有这个映射，你访问域名时浏览器不知道该去哪里找你的网站。

搬瓦工 VPS 本身不提供域名解析服务——它只是一台装着系统的服务器，有一个固定的 IPv4 地址。把域名"接"到这台服务器上，需要你自己去域名解析服务商那里操作。

整个建站流程大概是这样的：

1. 购买搬瓦工 VPS，拿到服务器 IP
2. 注册域名（在国外注册商如 Namesilo、Dynadot 等）
3. 在域名解析服务商（DNSPod 或 Cloudflare）添加 A 记录，把域名指向 VPS 的 IP
4. 在域名注册商处修改 Name Server，指向你用的解析服务商
5. 等解析生效，开始建站

**搬瓦工域名解析**这个流程，本质上是步骤 3 和 4 的组合操作。两步缺一不可。

## 第二步：选择域名解析服务商，添加 A 记录

主流的免费解析服务商就两个选择：**DNSPod**（国内）和 **Cloudflare**（国外）。

两者区别：

- **DNSPod**：国内用户解析速度快，支持按运营商分线路解析（电信、联通、移动走不同 IP），面向国内访客的网站适合用它
- **Cloudflare**：自带免费 CDN 和基础防攻击能力，网站被 DDoS 时可以一键开启防护；如果 VPS 线路本身够快，CDN 其实用不上，但防护能力是它的优势

搬瓦工 CN2 GIA-E 线路本身速度已经不错，Cloudflare 的 CDN 加速对它意义不大，但如果你担心攻击，Cloudflare 是更好的选择。如果就是普通建站，DNSPod 操作更简单，国内生效速度也快。

### 以 DNSPod 为例，添加 A 记录的步骤：

**1.** 注册并登录 DNSPod，点击"添加域名"，把你的域名填进去

**2.** 添加两条 A 记录：

| 主机记录 | 记录类型 | 记录值 |
|:---:|:---:|:---:|
| `@` | A | 你的搬瓦工 VPS IP |
| `www` | A | 你的搬瓦工 VPS IP |

`@` 对应裸域名（example.com），`www` 对应带 www 的域名（www.example.com）。两条都要加，TTL 用默认的 600 就行，不用改。

**3.** 保存后，DNSPod 会给你两个 DNS 服务器地址（NS 记录），记下来，下一步要用。

## 第三步：在域名注册商处修改 Name Server

这一步经常被忘掉，也是很多人解析半天没生效的根本原因。

添加了 A 记录还不够——你得告诉域名注册商，"我这个域名的解析，以后交给 DNSPod 来管"。这个操作叫修改 **Name Server（NS 服务器）**。

以 Dynadot 为例：

**1.** 登录 Dynadot 后台，进入域名管理

**2.** 找到 **DNS Settings** 或 **Name Server**，点击修改

**3.** 选择自定义 Name Server，把 DNSPod 给你的两个 DNS 服务器地址填进去

**4.** 保存，等生效

如果你用的是阿里云（万网）注册的域名：进入阿里云域名控制台 → 域名列表 → 点击域名 → 找到"DNS 修改"，把 NS 服务器改成 DNSPod 的即可。

不管哪个注册商，逻辑都一样：找到"DNS 服务器"或"Name Server"选项，改成你的解析服务商提供的 NS 地址。

## 解析生效需要多久，为什么等了好几天还没好

正常情况下，几分钟到半小时就能生效。但有时会拖到几个小时，极少数情况下要等到 24 小时。

生效慢的常见原因：

- TTL 之前被设置得很大（比如 86400，也就是 24 小时），改了记录要等旧缓存过期
- 本地 DNS 缓存还没刷新——在命令行里执行 `ipconfig /flushdns`（Windows）或者 `sudo dscacheutil -flushcache`（Mac）清一下
- Name Server 还没真正改好，可以用 [whatsmydns.net](https://www.whatsmydns.net) 检查你的域名在全球各地解析的情况

我自己用下来的感受是：DNSPod 解析通常 5-10 分钟就好了，Cloudflare 稍慢一点，不过也很少超过半小时。要是等了两三个小时还没生效，就得回头查 NS 有没有改对、A 记录的 IP 有没有填错。

## 机房切换后域名解析要重新配吗

这个问题挺常见的。答案是：**要更新 A 记录**。

搬瓦工在 KiwiVM 后台支持一键切换机房，但切换之后 **IP 地址会变**（IPv6 地址保持不变）。原来解析的 IP 就失效了，必须回到 DNSPod 或 Cloudflare，把 A 记录的值改成新的 IP，然后等解析生效。

新 IP 还是在 KiwiVM 首页看，切换完机房之后刷新一下就有。

所以如果切了机房之后网站打不开，第一个检查点就是 A 记录有没有更新。

## 反向 DNS（PTR 记录）和域名解析是两回事

自建邮件服务器的朋友可能会接触到这个词。正向 DNS 是域名 → IP，反向 DNS 是 IP → 域名。

搬瓦工 KiwiVM 后台支持设置 PTR 记录——在首页最下方找到 **PTR Records**，点击 "set new record"，填入你想绑定的域名就行。主要用途是发邮件时验证发件服务器的合法性，不设置的话发出去的邮件更容易进垃圾箱。

普通建站用不到反向 DNS，只有自建邮件服务器才需要配。

## 常见问题 Q&A

**Q：域名解析之后，还需要在搬瓦工后台做什么设置吗？**

A：不需要。搬瓦工 VPS 这边不用任何额外设置，IP 地址是你操作的全部。解析这件事完全在域名解析服务商和域名注册商两边完成。

**Q：国内域名（.cn）能用搬瓦工 VPS 建站吗，需要备案吗？**

A：理论上可以解析，但国内域名严格来说要配合国内服务器才需要备案，搬瓦工是境外 VPS，不在工信部备案体系里，.cn 域名绑到境外服务器通常是可以访问的，不会被强制要求备案。不过实际情况复杂，做正式站建议用 .com 或 .net 域名，更稳。

**Q：一台搬瓦工 VPS 可以绑多少个域名？**

A：没有数量限制。只要在域名解析服务商那边给每个域名分别添加 A 记录，都指向同一个 IP，然后在 VPS 的 Web 服务器（如 Nginx 或 Apache）里配置对应的虚拟主机，就能实现一个 IP 跑多个网站。

**Q：换了机房之后 IP 变了，A 记录更新完需要等多久才能访问？**

A：同样是几分钟到半小时，跟第一次解析一样。如果你把 TTL 设置得比较短（比如 600 秒），变更生效会更快。

**Q：搬瓦工限量版套餐可以用来建站做域名解析吗？**
